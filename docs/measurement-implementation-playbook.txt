Ovejitas Measurement Implementation Playbook

Purpose
- This file is a long-term implementation guide for measurement-related improvements.
- It is designed so we can continue in multiple sessions without losing context.
- Priority is useful farm outcomes over visual polish.

Project Context (for future sessions)
- App: Ovejitas (farm/sheep management frontend).
- Goal: improve measurement usefulness for health and operational decisions.
- Existing work already done:
  - Latest-per-type measurement endpoint integration.
  - Delta fields in UI (change and changePercent).
  - Trend direction visuals (up/down/neutral).
  - Basic anomaly highlighting on significant changes.
  - Measurement history modal responsive and scrollable.

Why we are doing this
- Farmers need fast, actionable health signals.
- Measurements are most valuable when they support decisions:
  - Who needs attention now?
  - Who is improving or declining?
  - What action should be taken next?
- We should avoid cosmetic-only features and focus on practical value.

Guiding Principles
- Use backend-filtered and paginated data, not large client-side filtering.
- Keep logic explainable (clear thresholds and reasons for alerts).
- Prefer features that reduce missed health issues and save time.
- Keep implementation incremental and safe (one feature at a time).

Recommended Implementation Roadmap

Phase 1 (highest value, low-medium effort)
1) Measurement Trend Charts (30/60/90 days)
- What:
  - Add charts for weight, height, and temperature over time.
  - Optional overlay with herd average for comparison.
  - Healthy/warning/critical zones when range data is available.
- Why:
  - Trend shape is easier to understand than table rows.
  - Helps detect gradual decline early.
- Notes:
  - Keep chart interaction simple for mobile.

2) Smart Measurement Alerts (rule-based)
- What:
  - Add practical alert rules, for example:
    - Rapid weight drop in short window.
    - Fever spike or persistent elevated temperature.
    - Large change percentages beyond threshold.
  - Show alert reason text, not only a badge.
- Why:
  - Faster triage and clearer action for farm owners.
- Notes:
  - Start with conservative thresholds and tune with real data.

Phase 2 (high usefulness for herd decisions)
3) Herd Measurement Summary View
- What:
  - Aggregate view with average/min/max by measurement type.
  - Count and percentage of animals in warning state.
  - Quick filters: all, needs attention, stable, improving.
- Why:
  - Enables farm-level decisions instead of one-animal-at-a-time review.

4) Body Condition Style Indicator (derived metric)
- What:
  - Add a derived status indicator from available measurements.
  - Keep it transparent: explain formula/criteria in UI.
- Why:
  - More practical than raw values alone for many husbandry decisions.

Phase 3 (advanced insights)
5) Growth Benchmark View by Age/Breed
- What:
  - Compare an animal trend against expected growth curve.
- Why:
  - Detect underperformance and optimize feeding/treatment.

6) Measurement + Event Correlation Timeline
- What:
  - Align measurements with events (treatments, diet changes, etc.).
- Why:
  - Helps identify what interventions worked.
