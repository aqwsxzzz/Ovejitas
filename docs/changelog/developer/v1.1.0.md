# v1.1.0 Developer Changelog

## Scope
- Release window: v1.0.0 -> v1.1.0
- Primary domains:
  - Auth routing/session guard behavior
  - Livestock flock detail UX and event-query behavior
  - Animal profile summary card presentation

## Architecture / Design Changes
- Flock-detail route now validates and carries a typed `eventType` search param.
- Event log filtering was wired to URL state so route state is shareable and preserves navigation context.
- Scroll behavior on in-page filter changes now uses router-level non-reset navigation to avoid jump-to-top UX regressions.
- Auth guard flows were tightened around root/public route transitions and logout/session invalidation paths.

## API and Data Contract Changes
- Frontend request contract now actively uses event list query `type` for flock event log filtering.
- Route consumers navigating into flock-detail now provide explicit search shape compatibility (`eventType`) to satisfy stricter typed route contracts.
- No backend schema migration introduced in this release.

## Implementation Notes
- Flock detail updates:
  - Added event type selector options with `all` passthrough behavior.
  - Bound selector changes to route search updates and non-reset scroll navigation.
  - Updated dependent flock-entry links/navigation call sites to include route search object.
- Mutation flow updates:
  - Livestock update actions shifted to query-invalidation-friendly hook usage.
- Auth updates:
  - Route/access control and session handling paths were hardened to reduce inconsistent redirect/logout behavior.

## Code Examples
```tsx
void navigate({
  replace: true,
  resetScroll: false,
  search: (previous) => ({
    ...previous,
    eventType: next === "all" ? undefined : next,
  }),
});
```

## Migrations / Operational Steps
- No database migration required.
- Frontend-only release with route typing implications:
  - Ensure all navigations to `/v2/production-units/flock/$unitId` include the expected search object.

## Risks and Mitigations
- Risk: Route typing propagation can break unrelated call sites when search schema changes.
  - Mitigation: Release included call-site updates for all known flock-entry points and full build validation.
- Risk: Event-type filter behavior depends on backend honoring the `type` query filter.
  - Mitigation: Frontend now passes `type` consistently; backend behavior should be verified in integration environments.

## Validation and Test Evidence
- Lint: Passed (warnings only, no errors).
- Build: Passed on `main` before tagging.
- Release smoke: Route and bundling checks completed via production build pipeline.

## Follow-ups

- Consider splitting large route-level typing changes into dedicated preparatory PRs to reduce cross-feature compile impacts.
