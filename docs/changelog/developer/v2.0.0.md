# v2.0.0 Developer Changelog

## Scope

The v2 release. Headline is end-to-end multi-currency financial tracking
(currency of record, no conversion, per-currency reporting), layered on the
reports, livestock-lifecycle, pregnancy, and settings work, plus removal of the
remaining legacy v1 frontend surfaces.

## Architecture / Design Changes

- **Currency of record.** Currency is a first-class per-farm resource. Monetary
  events and material purchases reference it by `currency_id`; the FE never
  converts or sums across currencies. Display code resolution goes through a
  single cached farm-currency list.
- **Per-currency reporting.** `profitability-full` and `cost-per-unit` are
  consumed as one row per `(asset, currency)`. Feed is valued in the currency of
  the backing purchases; `currency` is nullable for genuinely unpriced feed.
- **New `currency` feature module** (`src/features/currency/`) owns API, queries,
  types, the picker, sticky-default resolution, and the management UI.
- **Farm default currency demoted** to a soft entry/reporting fallback; no report
  reads it as an authority anymore.

## API and Data Contract Changes

- `EventRead` / `MaterialPurchaseRead`: `currency: string` → `currency_id: number | null`.
- Create paths accept optional `currency_id` (events, material purchase/sale,
  flock acquisition/sale, individual sale).
- `IProfitabilityFullRow` / `ICostPerUnitRow`: `currency` now `string | null`;
  `has_other_currency` removed; rows are per `(asset, currency)`.
- Currencies endpoint: `GET/POST/PATCH/DELETE /farms/{id}/currencies`; list
  supports `q`, `sort`, `page`, `page_size`, and a tri-state `archived` filter
  (omitted = all, so archived codes still resolve on historical records).

## Implementation Notes

- `useDefaultCurrencyId` resolves preselection as last-used (localStorage) →
  farm default → first enabled, derived during render (no effects).
- `CurrencySelectField` is controlled; the parent owns the value and its
  submit-time fallback and remembers the chosen ISO code.
- `formatCurrency` tolerates a null currency (plain-number formatting).
- Dashboard Neto KPI and the asset "Neto real" card render per-currency via the
  existing KPI-slider / collapsible-block patterns — no summing.

## Migrations / Operational Steps

- No FE data migration. Deploy the compatible backend with or ahead of this
  release; the currency-of-record and per-currency report contracts are required.

## Risks and Mitigations

- **Breaking BE contract dependency** (`currency_id`, per-currency reports).
  Mitigation: coordinated backend deploy; roll back FE to v1.4.0 only alongside a
  backend rollback.
- **Archived-currency display** relies on the list returning archived rows.
  Verified against the live OpenAPI (`archived` tri-state defaults to all).
