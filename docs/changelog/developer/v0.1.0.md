# v0.1.0 Developer Changelog

## Scope
Release `v0.1.0` spans platform-level frontend changes across data loading strategy, animal listing/search/filter behavior, measurement summaries, dashboard counters, expenses filtering UX, auth input normalization, and delivery workflow automation.

Primary PRs included:
- #9: animal server filtering and search behavior
- #10: measurement latest-summary, delta tracking, anomaly detection
- #11: dashboard counters from stats endpoint
- #12: expenses responsive filter redesign + auth email normalization
- #14: release workflow prompt and process hardening

## Architecture / Design Changes
- Shifted high-volume list views toward server-first pagination and filtering, reducing client-side overfetching and improving scalability.
- Standardized filter/search UX patterns toward explicit query-driven interactions (including debounced query support utility).
- Expenses feature now uses responsive filter surfaces by breakpoint:
  - mobile: bottom sheet + floating trigger
  - tablet: toolbar filter panel
  - desktop: sticky left filter rail
- Release operations formalized via prompt-driven workflow, with stricter version-resolution and curated release-note governance.

## API and Data Contract Changes
- Animals:
  - Adopted server-side search/filter semantics in animal API/query layer.
  - Consolidated list retrieval behavior around paged requests.
- Measurement:
  - Added/consumed latest-summary contracts for per-animal measurement insights.
  - Introduced delta and anomaly-oriented fields in measurement types.
- Dashboard:
  - Counters now sourced from stats endpoint instead of derived client-side list counting.
- Expenses:
  - Query contract supports improved filter parameterization and route-level integration.
- Auth:
  - Login/signup payload email is normalized at API boundary via `trim().toLowerCase()`.

## Implementation Notes
- New responsive expenses filter components introduced:
  - `expense-filter-fab.tsx`
  - `expense-filter-panel.tsx`
  - `expense-filter-sheet.tsx`
- Expenses route orchestration updated to support draft-and-apply filter flow and breakpoint-specific rendering.
- Animal and measurement query modules updated to support server-driven list/search and summary behaviors.
- Shared utility `use-debounced-value` added for query/input synchronization patterns.
- i18n resources updated in EN/ES for affected domains (animals, expenses, farm members, measurement modal).
- Workflow artifacts updated under `.github/prompts/` for commit/PR/release process consistency.

## Risks and Mitigations
- Risk: Query behavior regressions when moving from client filtering to server filtering.
  - Mitigation: Endpoint-scoped query modules updated together with route/component consumers.
- Risk: Responsive filter UX inconsistency across breakpoints.
  - Mitigation: Dedicated components by viewport role and explicit apply/clear interactions.
- Risk: Auth edge-cases from email casing differences.
  - Mitigation: Input-level keyboard hints and API-level normalization both applied.
- Risk: Release process drift across contributors.
  - Mitigation: Centralized release prompt with explicit safety and quality gates.
