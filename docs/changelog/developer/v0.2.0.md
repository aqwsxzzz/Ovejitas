# v0.2.0 Developer Changelog

## Scope
Release `v0.2.0` delivers three substantive frontend domains:
- flock management flows
- financial transaction migration for the former expenses feature
- farm member and invitation management wiring

Additional scope includes the animal modal sizing fix and release workflow/process updates merged into `development` before promotion to `main`.

Primary PRs included:
- #17: flock management foundation
- #18: flock UX and detail handling polish
- #19: animal modal sizing stabilization
- #20: financial transactions migration and farm members wiring
- #21: release PR from `development` to `main`

## Architecture / Design Changes
- Added a dedicated flock feature surface with its own API/query/types modules and page-level composition.
- Replaced the legacy expense domain contract with a leaner financial-transactions model aligned to backend endpoints and serializer output.
- Removed stale expense-only enums, fields, labels, and i18n keys that no longer exist in the backend contract.
- Converted farm members from a scaffold placeholder route into a real route backed by existing invitation/member query modules.
- Moved pending invitation filtering to backend query params instead of client-side post-filtering.

## API and Data Contract Changes
- Expenses frontend now consumes financial transaction endpoints and summary semantics instead of the deprecated expense contract.
- Financial transaction payloads/types are now constrained to the backend-supported fields only:
  - `type`
  - `amount`
  - `description`
  - `speciesId`
  - `date`
- Farm invitation response typing now includes backend serializer fields such as `createdAt`, `updatedAt`, and `expiresAt`.
- Farm member typing now includes backend serializer timestamps and is imported from the correct domain module.
- Invitation list retrieval now supports backend `status` filtering and uses it for pending invitations.

## Implementation Notes
- Added new flock route surfaces:
  - `src/routes/_private/_privatelayout/farm/$farmId/flocks.tsx`
  - `src/routes/_private/_privatelayout/farm/$farmId/flocks/$flockId.tsx`
- Added flock UI modules for cards, filters, stats, events, creation, and count updates.
- Reworked the expenses page into a financial transactions page while preserving list, pagination, summary, and modal flows.
- Deleted unused expense UI modules that depended on removed legacy contract fields.
- Added lightweight farm member/invitation list items for the members page route.
- Developer release documentation process introduced in the previous release is now exercised again with this artifact.

## Code Examples
Server-side invitation filtering instead of client-side post-filtering:

```ts
useGetFarmInvitationsList({ farmId, status: "pending" })
```

Financial transaction create payload shape:

```ts
{
  type,
  amount,
  description,
  speciesId,
  date,
}
```

## Migrations / Operational Steps
- No backend schema migration is required by this release from the frontend repository perspective.
- Frontend package version bumped from `0.1.0` to `0.2.0`.
- Recommended manual smoke checks after deploy:
  - open flock list and flock detail routes
  - create/edit/delete a financial transaction
  - open farm members page and verify pending invitations render

## Risks and Mitigations
- Risk: legacy frontend code paths may still assume removed expense fields.
  - Mitigation: dead code and stale type branches were removed during migration.
- Risk: invitation/member UI could drift from backend serializer shape.
  - Mitigation: FE types were tightened to current backend route/schema/serializer outputs.
- Risk: lint gate remains unreliable in local release execution.
  - Mitigation: build validation passed; lint failure is documented as environment/package-install related.
- Risk: release scope is broad because it aggregates all `development` work since `v0.1.0`.
  - Mitigation: release notes and this changelog are grouped by domain instead of raw commits.

## Validation and Test Evidence
- `git pull --ff-only origin main` passed during release preparation.
- `npm run build` should be used as the primary local release smoke gate and is expected to pass on this release commit.
- `npm run lint` remains environment-blocked when the local install is missing `eslint-plugin-react-you-might-not-need-an-effect`.
- Release PR #21 merged into `main` at commit `c580def97bc872aaff73942699468408eb7b311a` before version bump.
